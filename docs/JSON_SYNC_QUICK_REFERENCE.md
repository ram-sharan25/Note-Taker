## Key Code Snippets

### JSON Format
```json
{
  "id": "56C3785A-5616-424F-8A3E-99F3C2474332",
  "status": "DONE",
  "changed_at": "2026-03-02T12:35:00-05:00"
}
```

### Filename Convention
```
sync/<origin-id>_<new-state>_<timestamp>.json

Examples:
sync/56C3785A-5616-424F-8A3E-99F3C2474332_DONE_1709467200000.json
sync/F85F4DCD-9FA2-4022-B624-C390D7F4822C_IN-PROGRESS_1709467201234.json
```

### Build JSON
```kotlin
fun buildStateChangeJson(originId: String, newState: String, timestamp: Long): String {
    val iso8601 = Instant.ofEpochMilli(timestamp)
        .atZone(ZoneId.systemDefault())
        .format(DateTimeFormatter.ISO_OFFSET_DATE_TIME)
    
    return """
        {
            "id": "$originId",
            "status": "$newState",
            "changed_at": "$iso8601"
        }
    """.trimIndent()
}
```

### New updateTodoState() Flow
```kotlin
suspend fun updateTodoState(noteId: Long, newState: String): Result<Unit> = runCatching {
    val note = noteDao.getById(noteId) ?: throw IllegalArgumentException("Note not found")
    val oldState = note.todoState
    
    // 1. Optimistic DB update
    noteDao.updateState(noteId, newState)
    
    // 2. Write JSON
    val timestamp = System.currentTimeMillis()
    val json = buildStateChangeJson(note.headlineId, newState, timestamp)
    val filename = generateSyncFilename(note.headlineId, newState, timestamp)
    
    val writeResult = localFileManager.writeSyncJson(filename, json)
    if (writeResult.isFailure) {
        // Rollback
        noteDao.updateState(noteId, oldState ?: "TODO")
        throw writeResult.exceptionOrNull()!!
    }
    
    // 3. Toggl integration (non-blocking)
    handleTogglStateChange(note, oldState, newState)
}
```

## Directory Structure
```
<org-root>/
├── agenda.org                    # Generated by Emacs (M-m s e)
├── inbox.org                     # User's source files
├── Brain/
│   └── tasks.org
├── phone_inbox/                  # Mobile quick capture
│   └── *.org
└── sync/                         # NEW - JSON sync directory
    ├── <uuid>_<state>_<ts>.json  # Pending state changes
    └── failed/                   # Processed by Emacs
        ├── <uuid>_<state>_<ts>.json
        └── <uuid>_<state>_<ts>.error
```

## Migration Steps (User-Facing)

1. Update mobile app to 0.9.0
2. Update Emacs config with JSON sync functions
3. Run `M-m s e` to generate agenda.org with ORIGIN_ID
4. Test state change on mobile
5. Run `M-m s s` on Emacs to process
6. Verify sync works end-to-end

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Pending syncs badge won't clear | Run `M-m s s` in Emacs |
| State changes don't appear | Check if JSON was processed (deleted from sync/) |
| JSON in sync/failed/ | Check .error file, manually fix source org file |
| "Cannot write to sync directory" | Recreate storage directory selection in Settings |

## Performance Targets

- JSON write: < 100ms
- Pending sync count: < 50ms
- App startup (1000 syncs): < 2s
- UI responsiveness: No blocking

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Property name | ORIGIN_ID | Clear distinction from old SOURCE_ID |
| Filename format | `<id>_<state>_<ts>.json` | Sortable, human-readable |
| Sync directory | Root level: `<org-root>/sync/` | Simple, alongside other org files |
| Write behavior | Optimistic DB update | Immediate UI feedback |
| Error handling | Rollback on failure | Data consistency |
| Toggl timing | Immediate on mobile | User responsiveness |
