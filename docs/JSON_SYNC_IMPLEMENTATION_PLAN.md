# Agenda JSON Sync Implementation Plan

## Overview

Transform the agenda system from **direct file editing** to a **JSON sync workflow**, where:
1. Mobile app writes state changes as JSON files to `sync/` directory
2. Emacs processes JSON files (`M-m s s`) to update actual source org files
3. Mobile app uses `ORIGIN_ID` property for stable identification
4. Syncthing handles bidirectional file sync

**Status:** Planning Complete - Ready for Implementation

**Target Version:** 0.9.0

**Estimated Timeline:** 4 weeks

---

## Problem Statement

### Current Architecture Issues

**Current Flow:**
```
Mobile: Tap TODO → Update Room DB → Read org file → Update OrgNode tree → Write back to file → Re-sync DB
Emacs: Edit org file → Syncthing syncs → Mobile detects hash change → Re-parses file
```

**Problems:**
- ❌ Race conditions when mobile and Emacs edit simultaneously
- ❌ Mobile writes can corrupt org files if parser/writer has bugs
- ❌ Complex conflict resolution (file hash checking, retry logic)
- ❌ Mobile needs full org-mode writer implementation
- ❌ ~800 lines of complex file manipulation code
- ❌ No audit trail of mobile changes
- ❌ Difficult to debug when things go wrong

### New Architecture Benefits

**New Flow:**
```
Mobile: Tap TODO → Update Room DB (optimistic) → Write JSON to sync/ → Done
Emacs: M-m s s → Process JSONs → Update source files → Delete JSONs → Export agenda.org
Syncthing: Syncs agenda.org back → Mobile detects change → Re-reads display
```

**Benefits:**
- ✅ Mobile never writes to source org files (simpler, safer)
- ✅ Emacs is the single writer (canonical source of truth)
- ✅ JSON sync survives mobile app crashes/uninstalls
- ✅ Emacs can batch-process multiple changes
- ✅ Failed JSONs go to `sync/failed/` for debugging
- ✅ Clean audit trail (JSON files with timestamps)
- ✅ ~800 lines of code can be removed
- ✅ Easier to test and maintain

## Technical Design

### 1. Property Name Change: ORIGIN_ID

**Current:** Headlines use `SOURCE_ID` or `ID` properties
**New:** Headlines use `ORIGIN_ID` as primary identifier

**Migration Strategy:**
- Parser reads: `ORIGIN_ID` → `SOURCE_ID` → `ID` (backwards compatible)

### 2. JSON Sync File Format

**Filename Convention:**
```
sync/<origin-id>_<new-state>_<timestamp>.json
```

**Examples:**
```
sync/56C3785A-5616-424F-8A3E-99F3C2474332_DONE_1709467200000.json
sync/F85F4DCD-9FA2-4022-B624-C390D7F4822C_IN-PROGRESS_1709467201234.json
sync/757CB098-7BFF-48EF-953D-B4A4D2B3DD9A_TODO_1709467205678.json
```

**JSON Content:**
```json
{
  "id": "56C3785A-5616-424F-8A3E-99F3C2474332",
  "status": "DONE",
  "changed_at": "2026-03-02T12:35:00-05:00"
}
```

**Fields:**
- `id` (string): ORIGIN_ID of the headline (UUID format)
- `status` (string): New TODO state (TODO, IN-PROGRESS, WAITING, HOLD, DONE, CANCELLED)
- `changed_at` (string): ISO 8601 timestamp with timezone

**Validation Rules:**
- Filename MUST match pattern: `^[A-F0-9-]+_[A-Z-]+_\d+\.json$`
- `id` field MUST match filename prefix
- `status` MUST be valid TODO keyword
- `changed_at` MUST be valid ISO 8601 timestamp

package com.rrimal.notetaker.data.models

import java.time.Instant
import java.time.ZoneId
import java.time.format.DateTimeFormatter

/**
 * JSON format for state change sync
 * Written by mobile app, read by Emacs
 * 
 * Example filename: 56C3785A-5616-424F-8A3E-99F3C2474332_DONE_1709467200000.json
 * 
 * JSON content:
 * {
 *   "id": "56C3785A-5616-424F-8A3E-99F3C2474332",
 *   "status": "DONE",
 *   "changed_at": "2026-03-02T12:35:00-05:00"
 * }
 */
data class StateChangeSyncMessage(
    val originId: String,      // ORIGIN_ID from org headline
    val newState: String,      // TODO, DONE, IN-PROGRESS, etc.
    val timestamp: Long        // Epoch milliseconds
)

/**
 * Build JSON string for state change sync
 * Uses manual JSON construction to avoid kotlinx.serialization dependency
 * 
 * @param originId ORIGIN_ID property from org headline
 * @param newState New TODO state (TODO, DONE, etc.)
 * @param timestamp Epoch milliseconds
 * @return JSON string
 */
fun buildStateChangeJson(originId: String, newState: String, timestamp: Long): String {
    val iso8601 = Instant.ofEpochMilli(timestamp)
        .atZone(ZoneId.systemDefault())
        .format(DateTimeFormatter.ISO_OFFSET_DATE_TIME)
    
    return """
        {
            "id": "$originId",
            "status": "$newState",
            "changed_at": "$iso8601"
        }
    """.trimIndent()
}

/**
 * Generate sync filename from state change
 * Format: <origin-id>_<new-state>_<timestamp>.json
 * 
 * Example: 56C3785A-5616-424F-8A3E-99F3C2474332_DONE_1709467200000.json
 */
fun generateSyncFilename(originId: String, newState: String, timestamp: Long): String {
    return "$originId\_$newState\_$timestamp.json"
}

/**
 * Validate sync filename format
 * Returns true if filename matches expected pattern
 */
fun isValidSyncFilename(filename: String): Boolean {
    // Pattern: <UUID>_<STATE>_<TIMESTAMP>.json
    // UUID: 8-4-4-4-12 hex digits with hyphens (or without)
    // STATE: All caps with optional hyphens
    // TIMESTAMP: Digits only
    val pattern = Regex("^[A-F0-9-]+_[A-Z-]+_\\d+\\.json$")
    return filename.matches(pattern)
}
```

/**
 * Ensure sync/ directory exists at root of org files
 * Called on app startup and before first sync write
 * 
 * @return Result with Unit on success, error on failure
 */
suspend fun ensureSyncDirectoryExists(): Result<Unit> = withContext(Dispatchers.IO) {
    runCatching {
        val rootUri = baseDirectoryUri ?: throw IllegalStateException("Root directory not set")
        val rootDir = DocumentFile.fromTreeUri(context, rootUri)
            ?: throw IllegalStateException("Cannot access root directory")
        
        // Check if sync/ already exists
        val existingSyncDir = rootDir.findFile("sync")
        if (existingSyncDir != null && existingSyncDir.isDirectory) {
            Log.d(TAG, "Sync directory already exists")
            return@runCatching
        }
        
        // Create sync/ directory
        val syncDir = rootDir.createDirectory("sync")
            ?: throw IOException("Failed to create sync directory")
        
        Log.d(TAG, "Created sync directory: ${syncDir.uri}")
    }
}

/**
 * Write JSON file to sync/ directory
 * Overwrites if file already exists
 * 
 * @param filename Filename only (not full path), e.g., "ABC-123_DONE_1709467200000.json"
 * @param content JSON content as string
 * @return Result with Unit on success, error on failure
 */
suspend fun writeSyncJson(filename: String, content: String): Result<Unit> = withContext(Dispatchers.IO) {
    runCatching {
        // Validate filename format
        require(isValidSyncFilename(filename)) {
            "Invalid sync filename format: $filename"
        }
        
        // Ensure sync directory exists
        ensureSyncDirectoryExists().getOrThrow()
        
        // Write to sync/<filename>
        val fullPath = "sync/$filename"
        writeFile(fullPath, content).getOrThrow()
        
        Log.d(TAG, "Wrote sync file: $fullPath")
    }
}

/**
 * List all JSON files in sync/ directory
 * Used for debugging and showing "pending syncs" indicator
 * 
 * @return Result with list of filenames (no path prefix)
 */
suspend fun listSyncFiles(): Result<List<String>> = withContext(Dispatchers.IO) {
    runCatching {
        val rootUri = baseDirectoryUri ?: throw IllegalStateException("Root directory not set")
        val rootDir = DocumentFile.fromTreeUri(context, rootUri)
            ?: throw IllegalStateException("Cannot access root directory")
        
        val syncDir = rootDir.findFile("sync")
        if (syncDir == null || !syncDir.isDirectory) {
            return@runCatching emptyList<String>()
        }
        
        syncDir.listFiles()
            .filter { it.isFile && it.name?.endsWith(".json") == true }
            .mapNotNull { it.name }
            .sorted()
    }
}

/**
 * Count pending sync files in sync/ directory
 * Lightweight version of listSyncFiles() for UI indicators
 */
suspend fun countPendingSyncs(): Result<Int> = withContext(Dispatchers.IO) {
    runCatching {
        listSyncFiles().getOrElse { emptyList() }.size
    }
}
```

/**
 * Update TODO state for a note via JSON sync
 * 
 * New behavior:
 * 1. Optimistically update local database (immediate UI feedback)
 * 2. Write JSON sync file to sync/ directory
 * 3. Emacs processes JSON (M-m s s) to update source file
 * 4. Emacs exports agenda.org (M-m s e)
 * 5. Syncthing syncs agenda.org back to mobile
 * 6. Mobile detects change and re-reads (automatic hash check)
 * 
 * @param noteId Database ID of the note
 * @param newState New TODO state (TODO, IN-PROGRESS, DONE, etc.)
 * @return Result with Unit on success, error message on failure
 */
suspend fun updateTodoState(noteId: Long, newState: String): Result<Unit> = runCatching {
    Log.d(TAG, "=== updateTodoState START (JSON SYNC) ===")
    Log.d(TAG, "  noteId: $noteId, newState: $newState")
    
    // 1. Get note from database
    val note = noteDao.getById(noteId) 
        ?: throw IllegalArgumentException("Note not found: $noteId")
    
    Log.d(TAG, "  Note found:")
    Log.d(TAG, "    filename: ${note.filename}")
    Log.d(TAG, "    headlineId: ${note.headlineId}")
    Log.d(TAG, "    title: ${note.title}")
    Log.d(TAG, "    currentState: ${note.todoState}")
    
    val oldState = note.todoState
    
    // 2. Optimistically update local database
    //    This provides immediate UI feedback without waiting for Emacs
    database.withTransaction {
        noteDao.updateState(noteId, newState)
    }
    Log.d(TAG, "  Database updated optimistically")
    
    // 3. Build JSON sync message
    val timestamp = System.currentTimeMillis()
    val json = buildStateChangeJson(
        originId = note.headlineId,
        newState = newState,
        timestamp = timestamp
    )
    
    // 4. Generate filename: <origin-id>_<state>_<timestamp>.json
    val filename = generateSyncFilename(note.headlineId, newState, timestamp)
    
    Log.d(TAG, "  Writing JSON sync file: sync/$filename")
    Log.d(TAG, "  JSON content: $json")
    
    // 5. Write JSON to sync/ directory
    val writeResult = localFileManager.writeSyncJson(filename, json)
    if (writeResult.isFailure) {
        // Rollback optimistic update
        Log.e(TAG, "  Failed to write sync file, rolling back", writeResult.exceptionOrNull())
        database.withTransaction {
            noteDao.updateState(noteId, oldState ?: "TODO")
        }
        throw writeResult.exceptionOrNull() 
            ?: IOException("Failed to write sync file")
    }
    
    Log.d(TAG, "  Successfully wrote sync file")
    
    // 6. Handle Toggl integration (if enabled)
    //    This happens immediately on mobile, doesn't wait for Emacs
    handleTogglStateChange(note, oldState, newState)
    
    Log.d(TAG, "  State change complete, waiting for Emacs to process")
    Log.d(TAG, "=== updateTodoState END ===")
}
```

**Error handling:**
- If JSON write fails → Rollback database update
- If database update fails → Don't write JSON
- If Toggl fails → Log but continue (non-blocking)

```markdown
# Agenda JSON Sync Architecture

## Overview
Mobile app uses JSON files in `sync/` directory to communicate state changes to Emacs.

## Workflow
1. User taps TODO item on mobile
2. App writes JSON: `sync/<origin-id>_<new-state>_<timestamp>.json`
3. Syncthing syncs to desktop
4. Emacs processes: `M-m s s` → updates source files → deletes JSONs
5. Emacs exports: `M-m s e` → updates `agenda.org`
6. Syncthing syncs agenda.org back to mobile
7. Mobile detects change → re-displays

## JSON Format
```json
{
  "id": "56C3785A-5616-424F-8A3E-99F3C2474332",
  "status": "DONE",
  "changed_at": "2026-03-02T12:35:00-05:00"
}
```

## Directory Structure
```
<org-root>/
├── agenda.org              # Mobile reads this (generated by Emacs)
├── inbox.org               # Source files (Emacs edits)
├── sync/                   # Mobile writes here
│   ├── <uuid>_<state>_<ts>.json
│   └── failed/             # Emacs moves failed syncs here
│       ├── <uuid>_<state>_<ts>.json
│       └── <uuid>_<state>_<ts>.error
└── phone_inbox/            # Mobile quick capture
    └── *.org
```

## Error Handling
- Failed JSONs moved to `sync/failed/` by Emacs
- Mobile shows "Pending syncs" badge when JSONs exist
- No automatic retry (user must manually fix via Emacs)

## Emacs Integration
See Emacs configuration for:
- `M-m s s` (sync): Process JSON files
- `M-m s e` (export): Generate agenda.org
```
